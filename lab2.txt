Тестируемый продукт: интернет-магазин bukvaeshka.by
Тестовое окружение: десктопные и мобильные браузеры
---------------------------------------------------
        Тест-кейсы:
1.	Валидация строки поиска.
При вводе пустой строки пользователь должен быть проинформирован об этом.
	1) Пользователь нажимает на строку поиска.
	2) Пользователь нажимает клавишу "Enter" / кнопку с иконкой лупы.
	3) У пользователя должна отобразиться в строке надпись "Пожалуйста, введите поисковый запрос"

Ожидаемый/реальный(ручная проверка на сайте) результаты НЕ совпали:
	После шага 2) пользователя редиректит на страницу с пустым результатом поиска.
*******************
2.	Корректность работы добавления в "Отложенное".
Пользователь должен иметь возможность добавлять товар в "Отложенное"
	1) Пользователь в разделе "Конструктор" нажимает на иконку с сердцем на товаре "Конструктор деревянный Десятое королевство Танк, 6 элементов".
	2) Товар должен оказаться в категории "Отложенные" раздела "Корзина"

Ожидаемый/реальный(ручная проверка на сайте) результаты совпали.
*******************
3.	Валидация написания e-mail в подписке на новости
В e-mail недопустимо использовать символы отличные от латинский букв, цифр, "." и "@".
	1) Неавторизованный пользователь на главной странице под надписью "Будьте всегда в курсе!" в строке "Ваш e-mail" вводит "11我爱中国111@1111.11". 
	2) Пользователь должен быть переадресован на страницу bukvaeshka.by/personal/subscribe/
	3) Пользователь должен увидить надпись "Указан неверный адрес подписки."

Ожидаемый/реальный(ручная проверка на сайте) результаты совпали.
*******************
4.	Валидация написания телефона в форме "Заказать звонок"
В номере телефона недопустимо использовать латинские символы.
	1) Пользователь нажимает на кнопку "Заказать звонок" на главной странице сайта.
	2) Пользователю должна открыться форма для ввода данных. 
	3) Пользователь вводит "dmdmd" в строку "Ваше имя" и "dmdmd" в строку "Телефон".
	4) Пользователь нажимает на кнопку "Отправить".
	5) Пользователь должен увидить надпись "Указан неверный телефон."

Ожидаемый/реальный(ручная проверка на сайте) результаты НЕ совпали:
	После шага 4) пользователь видит на экране надпись "Ваше сообщение успешно отправлено."
*******************
5.	Корректность редиректа на аккаунт в Instagram
Официальный аккаунт располагается на сайте www.instagram.com
	1) Пользователь нажимает на иконку с логотипом Instagram в верхнем правом углу хедера сайта.
	2) В отдельной вкладке должна открыться страница https://www.instagram.com/bukvaeshka.by/

Ожидаемый/реальный(ручная проверка на сайте) результаты совпали.
*******************
6. 	Корректность перехода в раздел "Промокод".
Пользователь должен иметь возможность использовать промокоды.
	1) Пользователь нажимает на раздел "Промокод".
	2) Пользователь должен быть переадресован на страницу для введения промокода.

Ожидаемый/реальный(ручная проверка на сайте) результаты НЕ совпали:
	После шага 1) пользователь оказывается на странице с мини-игрой про космос.
*******************
7.	Валидация ценового диапазона в разделе "Книги".
Пользователь должен понимать, что он ввел что-то неверно, а не что товара с такой ценой пока нет в наличии.
	1) Пользователь в разделе "Книги" в окно в верхним ценовым диапазоном вводит "D101.3D9".
	2) Пользователь должен увидеть задизейбленую кнопку "Показать товар" и надпись "Указан неверный ценовой диапазон.".

Ожидаемый/реальный(ручная проверка на сайте) результаты НЕ совпали:
	После шага 1) пользователь видит только кнопку "Показать 0 товаров" и может нажать на нее.
*******************
8.	Корректность работы лимитов на товары.
Пользователь не может добавлять товаров больше, чем прописано в лимите.
	1) Пользователь добавляет в корзину книгу "Атлант расправил плечи (комплект из 3-х книг)".
	2) Нажимая на кнопку "+" пользователь доходит до числа большего 18, и должен не иметь возможности нажимать на кнопку "+".

Ожидаемый/реальный(ручная проверка на сайте) результаты НЕ совпали:
	После шага 1) пользователь может добавлять товары в неограниченном количестве, и строка "Сумма" динамически изменятся. Через пару секунд после прекращения нажатия кнопки "+" строка "Количество" возвращается к "17",
а "Сумма" пересчитывается.
*******************
9.	Корректность сохранения покупок
	1) Пользователь добавляет в корзину 2 книги "Атлант расправил плечи".
	2) Пользователь переходит в раздел "Рисование".
	3) Пользователь перезагружает страницу.
	4) У пользователя должны остаться 2 книги в корзине.

Ожидаемый/реальный(ручная проверка на сайте) результаты совпали.
*******************
10.	Корректность выставления звезд товару.
Неавторизованный пользователь не должен иметь возможность оценивать товар.
Пользователь должен иметь возможность менять количество звезд товару.
	1) Неавторизованный пользователь выбирает товар "Картина по номерам "Сказочная лошадь", A3"
	2) При попытке нажать на 3-ю звезду из 5 у пользователя должно открываться окно для авторизации. 

Ожидаемый/реальный(ручная проверка на сайте) результаты НЕ совпали:
	После шага 1) неавторизованный пользователь может поставить 3 звезды, но не может их убрать. При перезагрузке страницы количество звезд остается 3.
